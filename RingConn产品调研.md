# 一、产品概述
## 1.1 产品定位

民用级健康监测智能戒指，以**24 小时无感佩戴**为基础，通过核心生理数据采集与算法分析，为用户提供通俗化、精准化的健康状态监测与预警，打造轻量化个人健康管理终端。

## 1.2 产品核心目标

1. 硬件端：实现核心生理数据的高精度、低功耗采集，保证佩戴舒适性与数据传输稳定性；
2. 软件端：将原始数据转化为普通人易理解的健康指标，实现多场景健康预警，提供个性化健康建议；

## 1.3 用户画像分析

### 核心目标人群

在人生不同阶段，女性会依次经历或同时拥有多种健康需求：

- 不想怀孕 → 避孕
- 准备怀孕 → 备孕
- 已经怀孕 → 孕期
- 日常健康管理 → 健康/减脂/皮肤/情绪

所有用户均为 **20～45岁育龄女性**，核心围绕「女性生理周期」展开，不同场景下需求侧重点不同，且同一用户可在不同阶段切换场景。

#### 1. 避孕需求用户（22～40岁）

**核心诉求：**
通过生理数据精准识别安全期/易孕期，辅助自然避孕或避孕决策，避免意外怀孕

**关注数据：**
- 排卵日、易孕期/高风险期、周期规律、基础体温趋势

**关键场景：**
- 暂无生育计划，希望通过科学方式避孕（不依赖药物/器械）
- 周期不规律，无法通过传统日历法判断安全期
- 同时关注避孕与基础健康（如睡眠、压力），避免避孕影响身体状态

**行为特点：**
- 高频查看周期状态，对"风险提示"敏感度高
- 需求直接（明确是否安全），讨厌复杂操作
- 重视数据隐私，希望产品简洁、不打扰

#### 2. 备孕需求用户（25～35岁）

**核心诉求：**
精准预测排卵日与最佳受孕时机，监测身体状态是否适合受孕，提升怀孕成功率

**关注数据：**
- 排卵日、易孕期、基础体温、HRV、睡眠质量、压力水平、黄体期长度

**关键场景：**
- 计划怀孕（3～6个月备孕周期），希望科学高效备孕
- 备孕1年以上未成功，需要更精准的身体监测
- 职场女性（工作忙碌），需要"碎片化"备孕指导（如同房提醒、作息建议）

**行为特点：**
- 对数据精度要求极高，愿意长期佩戴戒指、记录相关信息
- 主动学习备孕知识，接受深度健康分析与个性化建议
- 关注"受孕概率""身体适配度"等决策类信息

#### 3. 孕期需求用户（25～38岁）

**核心诉求：**
监测孕期健康指标，保障自身与胎儿安全，获取孕期专属指导

**关注数据：**
- 孕周、预产期、静息心率、睡眠结构、体重增长趋势、胎动关联信号（如活动量）

**关键场景：**
- **孕早期（1～3个月）：** 监测身体稳定性，警惕异常信号
- **孕中期（4～6个月）：** 管理体重、睡眠，适配孕期运动
- **孕晚期（7～10个月）：** 关注心率、血压变化，为分娩做准备

**行为特点：**
- 对"风险预警""安全提示"需求强烈，依赖专业建议
- 定期查看孕周进展与胎儿发育关联信息
- 希望数据与产检报告互补，减少焦虑

#### 4. 健康管理达人用户（20～45岁）

**核心诉求：**
通过多维度数据追踪整体健康，结合周期特点优化生活方式（减脂、睡眠、压力管理等）

**关注数据：**
- HRV、静息心率、睡眠质量（深睡/REM占比）、活动量、热量消耗、体脂率（若支持）、周期与代谢关联

**关键场景：**
- **减脂塑形：** 结合周期阶段调整运动强度与饮食
- **职场压力管理：** 通过HRV、压力数据优化作息，缓解焦虑
- **睡眠改善：** 针对性提升深睡质量，解决经期失眠等问题

**行为特点：**
- 喜欢深度数据洞察（如趋势分析、周期与健康的关联）
- 愿意尝试个性化建议（如周期适配的运动计划）
- 长期使用健康科技产品，重视数据连续性与可追溯性

> [!warning] 补充说明
> - **四类用户并非完全割裂：** 例如"健康管理达人"可能同时有避孕需求，"备孕用户"也可能关注减脂，产品需支持**需求叠加**（而非模式切换）。
> - **核心共性：** 所有用户都需要"周期相关数据"（排卵日、经期），只是在不同场景下，数据的优先级与解读方式不同。


### 她们有什么健康问题？

#### 1. 生理周期不规律
- 经期不准，周期长度波动大
- 排卵期难以预测，影响备孕计划
- 经期不适症状（痛经、乳房胀痛、情绪波动）

#### 2. 压力与情绪管理问题
- 工作生活压力大，缺乏有效的压力监测手段
- 焦虑、抑郁等情绪问题难以察觉和干预
- 长期高压力状态导致免疫力下降、睡眠质量差

#### 3. 睡眠质量问题
- 睡眠不足（每天<7小时）
- 睡眠质量差（频繁觉醒、深睡不足）
- 睡眠作息不规律（熬夜、晚睡晚起）
- 睡眠呼吸暂停风险缺乏监测

#### 4. 健康数据碎片化
- 健康数据分散在多个APP中（经期APP、运动APP、睡眠APP）
- 缺乏整合分析，无法看到健康全貌
- 数据录入依赖手动，准确性和连续性差

#### 5. 个性化健康指导缺失
- 现有产品提供通用建议，未考虑个人生理特征
- 健康建议与生理周期脱节
- 缺乏针对不同健康目标（备孕、避孕、健康维持）的差异化指导

### 现有产品无法解决什么痛点？

#### 痛点1：手动记录为主，数据质量差
- **问题描述：** 现有经期APP依赖用户手动记录月经开始/结束日期、症状等
- **影响：**
	- 数据准确性依赖用户记忆和自律
	- 容易遗漏或延迟记录
	- 缺乏客观生理指标支撑（如体温、HRV变化）
	- 难以捕捉细微的生理变化

#### 痛点2：周期预测精度不足
- **问题描述：** 基于历史周期长度的简单统计推算
- **影响：**
	- 排卵期预测误差可达±3-5天
	- 对周期不规律用户几乎无效
	- 未考虑身体状态变化（如压力、睡眠、疾病对周期的影响）
	- 备孕用户错过最佳受孕窗口

#### 痛点3：数据维度单一，缺乏整合
- **问题描述：**
	- 经期APP只记录经期信息
	- 运动APP只记录运动数据
	- 睡眠APP只记录睡眠信息
- **影响：**
	- 用户无法看到健康全貌
	- 无法分析不同指标之间的关联（如压力→睡眠→经期的影响链）
	- 缺乏综合健康评估

#### 痛点4：健康建议与周期脱节
- **问题描述：** 提供的是通用健康建议，未考虑用户当前所处的生理周期阶段
- **影响：**
	- 建议缺乏时效性和相关性
	- 同样的运动建议在月经期和排卵期可能完全不适用
	- 无法利用不同周期阶段的生理特点进行针对性干预

#### 痛点5：个性化程度不足
- **问题描述：** 提供通用健康建议，未结合个人生理特征
- **影响：**
	- 建议缺乏针对性，无法满足个体差异需求
	- 难以根据用户独特的生理特点提供精准指导
	- 无法充分发挥健康数据的个性化价值
	- 用户体验和干预效果大打折扣

#### 痛点6：隐私安全顾虑
- **问题描述：**
  - 数据存储和传输可能存在安全漏洞
  - 用户对数据使用范围缺乏透明了解和控制权
- **影响：**
	- 用户担心隐私泄露
	- 不愿录入详细健康信息
	- 对产品信任度降低

## 3.2 传感器类型及工作机制

### 1. PPG光电容积描记传感器
**核心物理原理：朗伯-比尔定律（Lambert-Beer Law）**
光通过吸收介质时，其强度随介质厚度和浓度呈指数衰减：I = I₀ × e^(-ε×c×l)
其中：I₀为入射光强，I为透射光强，ε为摩尔吸光系数，c为浓度，l为光程长度。

**硬件构成：**
- 发光二极管（LED）：通常包含三种波长
  - 绿光（~525nm）：主要用于心率检测
  - 红光（~660nm）：用于血氧检测
  - 红外光（~880nm）：用于血氧检测
- 光电二极管（PD）：接收反射或透射的光线，转换为电信号
- 驱动电路：控制LED的发光强度和时序
- 信号调理电路：放大、滤波PD输出的微弱电信号

**工作机制：**
1. LED向皮肤发射特定波长的光线
2. 光线穿透皮肤表层，到达皮下血管
3. 血液中的血红蛋白（Hb）和氧合血红蛋白（HbO₂）对不同波长的光具有不同的吸收特性
4. 心脏收缩时，血管充血，血液容积增加，光吸收增加，光电二极管接收的光强减少
5. 心脏舒张时，血管收缩，血液容积减少，光吸收减少，光电二极管接收的光强增加
6. 这种周期性的光强变化形成PPG波形，通过检测波形的峰值间隔可计算心率

### 2. NTC负温度系数热敏电阻传感器
**核心物理原理：半导体材料的电阻率随温度变化的特性**
NTC热敏电阻由金属氧化物（如锰、钴、镍的氧化物）在高温下烧结而成，其电阻值随温度升高呈指数下降：
R(T) = R₀ × e^(B×(1/T - 1/T₀))
其中：R(T)为温度T时的电阻，R₀为参考温度T₀时的电阻，B为材料常数。

**硬件构成：**
- NTC热敏电阻芯片
- 高精度分压电路（与精密固定电阻串联）
- 模数转换器（ADC）：将电压信号转换为数字信号

**工作机制：**
1. NTC热敏电阻与皮肤紧密接触，达到热平衡
2. 皮肤温度变化导致NTC电阻值变化
3. 分压电路输出电压随电阻值变化而变化
4. ADC采集电压值，通过校准后的温度-电阻转换公式计算皮肤温度

### 3. MEMS电容式加速度计
**核心物理原理：牛顿第二定律（F = ma）**
物体的加速度与所受合力成正比，与质量成反比。通过测量惯性质量在加速度作用下产生的位移，可推算加速度。

**硬件构成（微观结构）：**
- 固定电极板（上、下各一个）
- 可移动电极板（中间，与惯性质量块连接）
- 弹性支撑结构（悬臂梁或弹簧）
- 电容检测电路
- 微型封装

**工作机制：**
1. 当传感器受到加速度时，惯性质量块由于惯性产生位移
2. 可移动电极板随质量块移动，改变与固定电极板之间的距离
3. 电容值C = ε₀×A/d，其中ε₀为真空介电常数，A为极板面积，d为极板间距
4. 上电容和下电容的差值变化反映加速度的大小和方向
5. 通过检测三轴（X、Y、Z）的电容变化，可获得三维加速度向量

### 4. MEMS陀螺仪
**核心物理原理：科里奥利效应（Coriolis Effect）**
当一个旋转物体同时受到垂直于旋转轴的加速度时，会产生科里奥利力：F_c = 2m×ω×v
其中：m为质量，ω为旋转角速度，v为线速度。

**硬件构成：**
- 驱动质量块（通过静电或电磁驱动做高频振动）
- 检测质量块（与驱动质量块耦合）
- 驱动电极和检测电极
- 弹性支撑结构

**工作机制：**
1. 驱动电路使驱动质量块沿X轴做简谐振动
2. 当传感器绕Z轴旋转时，检测质量块受到科里奥利力作用沿Y轴产生位移
3. 检测质量块的位移导致检测电容变化
4. 通过检测电容变化的幅度和相位，可计算角速度

## 3.3 数据传输机制

智能戒指本身不直接连云端，它的通信能力非常有限。数据传输的完整链路是这样的：

- **戒指端：**
内置的传感器（PPG、加速度计、陀螺仪等）采集原始数据。
这些数据先存储在戒指内部的Flash 闪存中，作为临时缓存。
戒指通过 ** 蓝牙（BLE）** 模块，与手机 APP 建立连接，将缓存的数据同步到手机。

- **手机端：**
手机 APP 接收到数据后，会先在本地数据库做缓存，以保证离线查看和快速响应。
当手机连接到互联网（Wi-Fi 或移动网络）时，APP 会将数据上传到厂商的云端服务器进行永久存储和分析。

- **云端：**
数据在云端进行存储、处理和算法分析（如睡眠分期、压力评估等）。
当你在 APP 中查看历史报告或趋势图时，APP 会从云端拉取数据并展示。

```
┌─────────────┐
│  智能戒指   │ → BLE同步 → ┌─────────────┐
│  (数据采集)  │             │   手机APP    │ → 本地计算
└─────────────┘             │  (端侧处理)  │ → 离线可用
```

### 生理基线水平确定（P0）

#### 生理基线校准规则

1. **基线定义**
    
    基于用户**连续 7 天以上健康状态**下的生理数据，计算个人静息心率、HRV、皮肤温度等指标的均值与正常波动范围。
    
2. **健康状态判定条件**
    
    - 生理指标长期稳定、无显著异常波动
    - 睡眠结构正常
    - 无用户主动上报不适、感染、服药等情况
    - 多指标无同步异常偏离
3. **异常状态区分**
    
    - **短期波动**：单一指标小幅偏离，短期内恢复 → 不更新基线
    - **亚健康 / 疲劳**：多指标轻度异常，持续≤2 天 → 标记状态，不更新基线
    - **生病 / 感染可能**：多指标显著异常且持续≥3 天 → 标记为异常状态，**不纳入基线计算**
4. **基线更新策略**
    
    仅在用户**确认回归长期健康状态**后，才重新迭代基线，避免用生病 / 亚健康数据污染个人正常参考值。

### 指标采集与计算（P0）
#### 原子指标

##### 心率（HR）
- **指标内涵**
`HR`是指在单位时间内，心脏完成一次完整的收缩-舒张周期的次数。它是衡量心脏活动水平、心血管健康状态和疾病风险的重要指标，可反映运动强度、情绪状态、睡眠质量等多种生理状态。

- **数据来源**
PPG传感器的绿光波形信号

- **计算方法**
1. **信号预处理：**
   - 对PPG原始信号进行低通滤波（截止频率~10Hz），去除高频噪声
   - 进行高通滤波（截止频率~0.5Hz），去除基线漂移
   - 使用自适应滤波算法减少运动伪影

2. **峰值检测：**
   - 采用局部最大值搜索算法，在PPG波形中识别收缩期峰值
   - 设置自适应阈值，排除噪声干扰产生的伪峰值
   - 验证峰值间隔的合理性（正常心率范围：40-200 BPM）

3. **心率计算：**
   - 测量连续两个峰值之间的时间间隔（PP间期，近似等于心电的RR间期）
   - 心率 = 60 / 平均PP间期（单位：BPM，次/分钟）
   - 使用滑动窗口（通常5-10秒）进行平滑处理

##### 心率变异性（HRV）
- **指标内涵**
`HRV`是指心率变异性，即连续心跳间期的微小差异和波动程度。它反映了自主神经系统（交感神经和副交感神经）对心脏的调节能力，是评估压力水平、心血管健康、情绪状态的重要指标。

- **数据来源**
PPG传感器提取的PP间期序列（近似等于心电的RR间期序列）

- **计算方法**
**时域分析指标：**

1. **SDNN（全部窦性心搏间期标准差）：**
   - 公式：SDNN = √[(1/N) × Σ(RRi - RRmean)²]
   - 其中：N为心搏次数，RRi为第i个RR间期，RRmean为平均RR间期
   - 反映整体心率变异性，与自主神经总活性相关

2. **RMSSD（相邻RR间期差值的均方根）：**
   - 公式：RMSSD = √[(1/(N-1)) × Σ(RRi+1 - RRi)²]
   - 主要反映副交感神经（迷走神经）活性

3. **pNN50（相邻RR间期差值>50ms的百分比）：**
   - 公式：pNN50 = (NN50 / (N-1)) × 100%
   - 其中：NN50为差值>50ms的相邻间期数量
   - 也是副交感神经活性的指标

**频域分析指标：**
1. 对RR间期序列进行重采样（通常4Hz）
2. 应用快速傅里叶变换（FFT）或自回归模型（AR）进行功率谱分析
3. **LF（低频成分，0.04-0.15Hz）：** 反映交感和副交感神经的混合活性
4. **HF（高频成分，0.15-0.4Hz）：** 主要反映副交感神经活性，与呼吸相关
5. **LF/HF比值：** 反映交感-副交感神经平衡

##### 血氧饱和度（SpO₂）
- **指标内涵**
`SpO₂`是指动脉血氧饱和度，即血液中氧合血红蛋白（HbO₂）占全部血红蛋白的百分比。它反映了人体的氧合状态和呼吸功能，是评估心肺健康、睡眠呼吸暂停的重要指标。

- **数据来源**
PPG传感器的红光和红外光波形

- **计算方法**
**核心物理原理：** 氧合血红蛋白（HbO₂）和还原血红蛋白（Hb）对红光和红外光的吸收差异
- HbO₂对红光吸收少，对红外光吸收多
- Hb对红光吸收多，对红外光吸收少

1. **AC/DC分离：**
   - 对红光和红外光的PPG信号，分别分离直流分量（DC）和交流分量（AC）
   - DC分量：组织、骨骼、静脉血等对光的吸收（相对恒定）
   - AC分量：动脉血搏动引起的光吸收变化（随心跳周期性变化）

2. **计算R值：**
   - R = (AC_red / DC_red) / (AC_ir / DC_ir)
   - 其中：AC_red为红光AC分量，DC_red为红光DC分量，AC_ir为红外光AC分量，DC_ir为红外光DC分量

3. **SpO₂校准：**
   - 使用经验公式：SpO₂ = A - B×R（A、B为通过临床实验确定的校准系数）
   - 典型值：当R=1时，SpO₂≈97%；当R=0.4时，SpO₂≈100%；当R=3时，SpO₂≈70%

##### 皮肤温度
- **指标内涵**
皮肤温度是指人体表面皮肤的温度，它反映了人体的体温调节状态、代谢水平和环境适应能力。在女性健康领域，睡眠期间的基础体温变化是识别生理周期、预测排卵的关键指标。

- **数据来源**
NTC温度传感器的电压读数

- **计算方法**
1. **ADC转换：** 将NTC分压电路的模拟电压转换为数字值
2. **电阻计算：** 根据分压电路公式计算NTC的当前电阻值
3. **温度转换：** 使用Steinhart-Hart方程计算温度：
   - 1/T = A + B×ln(R) + C×(ln(R))³
   - 其中：T为绝对温度（K），R为NTC电阻，A、B、C为通过校准确定的系数
4. **温度补偿：** 考虑环境温度和佩戴状态进行补偿修正

##### 呼吸频率
- **指标内涵**
呼吸频率是指单位时间内呼吸的次数，它反映了呼吸系统的功能状态、代谢需求和自主神经调节。异常的呼吸频率可能提示呼吸障碍、心肺疾病或情绪异常。

- **数据来源**
PPG传感器的绿光波形（通过呼吸调制分量间接测量）

- **计算方法（基于PPG）**
1. **信号提取：** 从PPG信号中提取呼吸调制分量
   - 呼吸会引起胸腔起伏，间接影响静脉回流和动脉血压
   - 这种调制作用会反映在PPG信号的基线、振幅或频率上

2. **信号处理：**
   - 对PPG信号进行0.1-0.5Hz的带通滤波（正常呼吸频率：12-30次/分钟 = 0.2-0.5Hz）
   - 或使用信号分解算法（如小波变换、EMD）分离呼吸分量

3. **频率计算：**
   - 检测呼吸波形的峰值
   - 测量连续峰值间隔，计算呼吸频率

##### 手机 GPS 定位
获取用户位置，用于获取对应位置的天气情况。
##### 气温、雾霾、PM2.5等天气情况

通过气象查询等 API 工具进行获取。

#### 健康指标

##### 睡眠分期

- **指标内涵**
睡眠分期是指将睡眠过程划分为不同的阶段（清醒期、浅睡期、深睡期、REM期）。不同睡眠阶段具有不同的生理意义，深睡期主要用于身体修复，REM期主要用于记忆巩固和情绪调节。
**在 REM 睡眠阶段尚有躯体抽动、眼球快速运动及血压升高、心率加快、呼吸加快而不规则等间断的阵发性表现**。若在此期间被唤醒，大部分人会诉说正在做梦，并且REM 睡眠中的上述阵发性表现可能与梦境有关。

**正常睡眠是 NREM 睡眠和 REM 睡眠两个不同时相周期性交替的过程。** 入睡后，一般先进入NREM 睡眠 N1 期，随后相继过渡到 N2 和 N3 期，持续 80～120 分钟后转入 REM 睡眠，REM 睡眠持续 20～30 分钟后又转入 NREM 睡眠，在整个睡眠过程中 NREM 睡眠和 REM 睡眠两个时相有 4～5次交替。越接近清晨，NREM 睡眠深度越浅，REM 睡眠持续时间越长。两个时相的睡眠均可直接转为觉醒状态，但由觉醒转为睡眠则通常先进入 NREM 睡眠，而不能直接进入 REM 睡眠。

- **数据来源**
根据以下原子指标计算而来：活动强度、心率、心率变异性（HRV）、皮肤温度

- **计算方法**
1. **特征提取：**
   - 加速度特征：体动次数、平均加速度、零交叉率
   - 心率特征：平均心率、心率趋势
   - HRV特征：SDNN、RMSSD、LF/HF比值
   - 体温特征：相对体温变化

2. **睡眠/清醒判断：**
   - 当体动次数低于阈值，且心率下降，判断为入睡
   - 当体动次数超过阈值，且心率上升，判断为清醒

3. **睡眠分期（基于多特征融合）：**
   - 清醒期：体动频繁，心率较高且不稳定
   - 浅睡期：体动较少，心率开始下降，HRV有所增加
   - 深睡期：体动极少，心率最低且稳定，HRV较高
   - REM期：体动较少但有眼球运动，心率轻度升高且不稳定，HRV变化

##### 睡眠质量
- **指标内涵**
睡眠质量是对睡眠整体好坏的综合评估，它反映了睡眠的充足性、连续性和结构合理性。良好的睡眠质量对于身体恢复、记忆巩固、情绪调节和免疫力维持至关重要。

- **数据来源**
根据以下原子指标和复合指标计算而来：睡眠分期、活动强度、心率

- **评价方法**

- 由 3 大块加权合成（你给的指标我全部包进去）

 推荐权重（非常合理、行业通用）

1. **睡眠时长 35%**
2. **睡眠结构（深睡 + REM + 周期）占比 40%**
3. **睡眠连续性（觉醒次数、觉醒时长）25%**

> [!info] 睡眠质量等级分段

> 
> - 睡眠质量评分 ≥90%
> - 深睡、REM 占比正常
> - 觉醒极少、几乎一觉到天亮

> 
> ### 2. 良好 —— **80～89 分**
> 
> - 睡眠质量评分 85%～90%
> - 结构完整，轻微觉醒不影响恢复
> 
    > **评价语：睡眠质量良好，身心状态稳定**
> 
> ### 3. 一般 —— **70～79 分**

> - 睡眠质量评分 80%～85%
> - 可能入睡偏慢、或夜间醒 1～2 次
> - 深睡略不足
> 
    > **评价语：睡眠基本充足，可小幅改善**
>

>

> - 夜间多次醒、浅睡偏多、深睡不足
>

##### 压力指数
- **指标内涵**
压力指数是对心理和生理压力水平的量化评估，它反映了自主神经系统的平衡状态和身体的应激程度。长期高压力状态会导致心血管疾病、免疫力下降、情绪障碍等健康问题。

- **数据来源**
根据以下原子指标计算而来：心率变异性（HRV）、心率、活动强度、皮肤温度

- **计算方法**
1. **特征输入：**
   - HRV指标：SDNN、RMSSD、LF、HF、LF/HF比值
   - 心率：当前心率、心率趋势
   - 体动：加速度幅值
   - 体温：相对体温变化

2. **压力模型：**
   - 基于HRV的压力评估核心原理：
     - 压力升高时，交感神经兴奋，副交感神经抑制
     - 表现为：SDNN降低、RMSSD降低、LF/HF比值升高、心率升高
   - 使用机器学习模型（如回归模型、神经网络）将多特征融合为压力分数
   - 压力分数范围：0-100，数值越高表示压力越大

3. **全天压力分布：**
   - 按时间窗口（通常5-15分钟）计算压力分数
   - 统计一天中压力的变化趋势
   - 识别高压时间段


##### 活动强度

**一、核心计算指标**

活动强度仅采用以下**3 项核心指标**，满足商用精度要求：

1. 加速度（活动度 / 体动）
2. 实时心率
3. 静息心率（基值）

**二、计算逻辑**

活动强度以**代谢当量（MET）**为本质依据，通过**加速度 + 心率**融合计算，最终划分为**4 个等级**：静坐 / 低活动、轻度活动、中度活动、高强度活动。

1. 加速度：判断是否处于运动状态

- 加速度幅度小 → 静止 / 低活动
- 加速度幅度大 → 运动状态

2. 心率储备比例：判断运动强度等级

$(心率储备比例 = \frac{实时心率 - 静息心率}{最大心率 - 静息心率})$

- ＜30%：低强度
- 30%～59%：中等强度
- ≥60%：高强度

最大心率估算：

\(最大心率 ≈ 220 - 年龄\)


3. 多源融合判断

同时结合加速度与心率储备比例，避免单一指标误判：

- **仅体动、心率无明显上升 → 低强度**
- **体动明显 + 心率显著上升 → 中高强度**

##### 规律作息评分

- **指标内涵**
规律作息评分是对用户日常作息规律性的量化评估，反映生物钟稳定度。规律作息对于女性健康尤为重要，可影响内分泌平衡、生理周期规律性、睡眠质量和整体代谢水平。

- **数据来源**
根据以下原子指标计算而来：睡眠入睡时间、睡眠起床时间、睡眠时长稳定性

- **计算方法**
1. **作息规律性计算：**
   - 入睡时间标准差：最近7天入睡时间的波动程度，标准差越小越规律
   - 起床时间标准差：最近7天起床时间的波动程度，标准差越小越规律
   - 睡眠时长：每日睡眠时长与个人目标时长的偏差度

2. **综合评分模型：**
   - 作息规律性权重（60%）：基于入睡/起床时间标准差
   - 睡眠时长达标率权重（30%）：睡眠时长是否在7-9小时范围内
   - 周末与工作日差异权重（10%）：避免熬夜-补觉的恶性循环

3. **评分等级：**
   - 优秀（85-100分）：作息高度规律，生物钟稳定
   - 良好（70-84分）：作息基本规律，偶有小波动
   - 一般（50-69分）：作息不够规律，需要改善
   - 需改善（<50分）：作息严重不规律，建议调整

##### 体征状态

- **指标内涵**
体征状态是对整体生理健康状态的综合评估，整合多个关键生理指标，提供全面的身体状态画像。这是女性健康管理的基础指标，可反映身体是否处于良好状态。

- **数据来源**
根据以下原子指标计算而来：静息心率、HRV、皮肤温度、血氧饱和度、活动量、睡眠质量

- **计算方法**
1. **单指标健康评分：**
   - 静息心率评分：60-75 BPM为最佳，超出范围扣分
   - HRV评分：SDNN越高评分越高，反映自主神经功能
   - 体温评分：36.1-37.0℃为正常范围
   - 血氧评分：≥95%为良好，低于93%需关注
   - 活动量评分：日均活动量达标情况

2. **体征状态综合指数：**
   - 权重分配：静息心率(20%)、HRV(25%)、体温(15%)、血氧(15%)、活动量(15%)、睡眠(10%)
   - 综合评分范围：0-100分
   - 趋势分析：对比近期体征状态变化趋势

3. **健康状态分级：**
   - 优秀（85-100分）：各项指标良好，身体状态佳
   - 良好（70-84分）：基本健康，个别指标有提升空间
   - 一般（50-69分）：需要关注，建议针对性改善
   - 需改善（<50分）：多项指标异常，建议咨询医生

#### 用户基础信息
性别、身高、体重、孕周、第一次月经时间等。

#### 用户行为习惯

1. **佩戴习惯**：日均佩戴时长、佩戴时段、佩戴连续性
2. **活动习惯**：日常活动量、活动时段、运动强度分布
3. **睡眠习惯**：入睡时间、起床时间、睡眠时长、睡眠节律稳定性

### 健康筛查与预警（P0）
#### 心律失常筛查

聚焦普通人日常易出现、且能通过心率数据识别的**常见心律失常相关异常**，以**预警心率的异常节律 / 数值**为核心，主要包括：

1. **心动过速 / 心动过缓预警**
    
    实时监测静息 / 活动状态下的心率，当心率持续超出个人正常基线（如静息心率突然＞100 次 / 分、＜50 次 / 分），或短时间内骤升骤降，触发预警，这是心律失常最基础的信号；
    
2. **心率不齐（心律不齐）初步筛查**
    
    通过采集心跳间期（RR 间期，HRV 的核心数据），识别心跳的间隔是否规律，若出现**无规律的心跳间隔、突发的漏跳 / 早搏**等特征，判定为 “心律不齐” 并预警，覆盖室性早搏、房性早搏等常见早搏类型的初步识别；
    
3. **阵发性心率异常预警**
    
    针对非持续性的心律失常（如偶尔的早搏、短时间心动过速），通过持续监测（尤其是夜间睡眠时，身体静息状态下异常更易捕捉），记录异常发生的时间、频次，当频次超出阈值时触发预警，实现**早期苗头性信号的捕捉**。

以上均为**民用级的异常识别**，不会标注具体的心律失常疾病名称（如房颤、室上速），仅提示 “心率异常 / 心律不齐”，避免医疗级诊断的风险。
#### 感染早期预警

身体受到细菌 / 病毒感染时，**免疫系统会率先启动防御反应**，这个阶段身体会出现**细微的生理性变化**，是感染早期的典型生理信号：

1. **皮肤温度微升高**：感染后身体的炎症反应会让核心体温先出现**0.2-0.5℃的微升**（未达到发烧的 37.3℃阈值），手指皮肤温度会同步变化，智能戒指可精准捕捉这一细微波动；
2. **静息心率异常升高**：免疫系统工作会增加身体代谢负荷，导致静息心率**比个人基线高 5-10 次 / 分**（即使没有活动，心率也莫名偏快）；
3. **心率变异性（HRV）骤降**：感染引发的身体炎症会干扰自主神经系统，让 HRV 指标快速降低（身体调节能力下降，是身体处于 “应激状态” 的重要信号）；
4. **睡眠中生理指标紊乱**：感染早期的生理性变化在睡眠时会更明显（身体无活动干扰），比如睡眠中皮温持续微升、心率忽快、HRV 偏低，这也是智能戒指的监测优势。

以上信号**单独出现可能无意义**，但**组合出现时**，就是感染早期的强预警信号，智能戒指的算法核心就是识别这类**多指标协同异常**。

#### 头痛预警

 一、智能戒指能预警头痛的**核心生理依据**

1. **自主神经紊乱**：表现为**HRV 骤降、静息心率异常升高 / 波动**，这是头痛最核心的前兆 —— 偏头痛、紧张性头痛的发作，多与交感 / 副交感神经失衡相关，神经紊乱会先反映在心率和 HRV 数据上；
2. **皮肤温度 / 外周循环异常**：偏头痛患者发作前，常出现**手指 / 头部皮温骤升 / 骤降**（血管收缩 / 扩张的外在表现），智能戒指采集的手指皮温能直接捕捉这一变化，同时伴随外周循环状态变差；
3. **睡眠 / 身体状态异常**：熬夜、睡眠质量差、身体疲劳是头痛的重要诱因，戒指监测的**晨起活力低、身体累度高、睡眠碎片化**，会作为**头痛发作的风险预警信号**；
4. **身体应激状态持续**：身心松弛度低、日常负荷高（如持续紧张、轻微压力），会让身体长期处于应激状态，这是紧张性头痛发作的重要前兆，戒指能通过多指标持续监测这一状态。

以上信号**单一出现可能是普通身体不适**，但**组合出现 + 持续异常**，就是头痛发作的强前兆，算法核心就是识别这类 “前兆组合信号”。

二、**最适配的头痛类型**

智能戒指的头痛预警**并非对所有头痛都有效**，对**有明确生理前兆、与身体状态相关的原发性头痛**预警效果最好，对继发性头痛仅能做 “风险提示”，具体区分：

✅ **高适配（预警准确率高）**

1. 偏头痛：发作前生理前兆（神经紊乱、血管收缩 / 扩张）最明显，是智能戒指预警的核心适配类型；
2. 紧张性头痛：多由疲劳、紧张、睡眠差引发，诱因和前兆均能被戒指监测，预警效果好；
3. 劳累性头痛：由身体过度负荷、睡眠不足引发，戒指能通过 “累度、睡眠、心率” 精准捕捉诱因，提前预警。

❌ **低适配（预警效果有限）**

1. 继发性头痛：由感冒、高血压、颈椎病、脑部疾病等引发的头痛，核心诱因是原发病，戒指仅能捕捉身体的异常信号，但无法关联原发病，仅能做 “身体异常提示”，而非针对性头痛预警；
2. 突发性头痛：无任何生理前兆的突发性头痛（如脑血管突发问题引发），因无前期数据异常，戒指无法预警。

### 女性生理期监测

- **指标内涵**
女性生理期指标包括生理周期识别、排卵预测、经期管理等内容。通过连续监测生理指标变化，可以准确识别月经周期的不同阶段（月经期、卵泡期、排卵期、黄体期），为女性健康管理、备孕指导、避孕规划提供科学依据。

- **数据来源**
根据以下原子指标计算而来：皮肤温度、心率变异性（HRV）、睡眠质量，结合用户手动记录

- **计算方法**
1. **基础体温（BBT）提取：**
   - 从睡眠期间的体温数据中提取最低体温值（通常在深睡期）
   - 进行体温平滑处理，去除单日波动（如采用移动平均法）

2. **周期识别：**
   - 观察BBT变化模式：
     - 月经期：体温较低
     - 卵泡期：体温保持在较低水平
     - 排卵日：体温小幅下降，随后显著升高（0.3-0.5℃）
     - 黄体期：体温维持在较高水平
   - 结合HRV变化：黄体期副交感神经活性相对增加，HRV有所升高

3. **预测模型：**
   - 基于历史周期数据建立个人化预测模型
   - 融合多源数据：BBT、HRV、睡眠模式、周期长度历史
   - 使用时间序列预测算法（如ARIMA、LSTM）预测下次月经来潮日期和排卵期

> [!info]- 产品解决方案
> 
 > **一、个体特征初始化（仅 1 次）**
> 
> 1. 最近 1 次经期起始日（必填，精确到日）
> 2. 过往周期范围 / 是否规律（可选）
> 3. 特殊状态：备孕、服药、多囊等（可选）
> 
> 二、**周期内轻量校准（每周期 1 次，≤10 秒）**
> 
> - 算法预测今日来月经，弹窗让用户**确认 / 修改实际经期开始日期**。
> - 算法提示可能进入排卵期，用户一键反馈**有无排卵症状**（白带拉丝、腹痛等）。
> - 预测为经前期，用户一键记录**有无 PMS**（乳房胀痛、情绪差等）。
> 
> 三、**异常场景触发校准（按需出现）**
> 
> - 体温 / 心率数据连续异常偏高
> 
    > → 弹窗询问：**感冒 / 发烧 / 熬夜 / 运动 / 其他**，标记为干扰数据，不参与周期计算。
> 
> - 周期比平时**推迟 / 提前超过 7 天**
> 
    > → 主动确认：**压力大 / 生病 / 服药 / 备孕 / 时差 / 其他**，重新校准模型。
> 
> - 用户长期佩戴但数据突变
> 
    > → 提示：**是否更换佩戴方式 / 手指 / 设备异常**，排除非生理因素。
> 
> - 用户记录**怀孕、停经期、妇科治疗**
> 
    > → 自动暂停周期预测，切换为对应模式。

## 4.4 技术方案选择
### 1. 整体技术方案设计

针对智能戒指采集的生理指标（心率、HRV、呼吸频率、皮温、血氧）识别睡眠分期和女性生理周期，推荐采用**时序特征提取 + 分类模型**的端到端方案，核心思路是：

1. **数据预处理**：处理时序数据的缺失值、异常值，按时间窗口（如 5 分钟 / 30 分钟）聚合特征；
2. **特征工程**：提取时序统计特征（均值、方差、趋势）、时域 / 频域特征（针对 HRV）；
3. **模型训练**：分场景选择适配的模型，兼顾精度和部署可行性（智能设备端需轻量化）；
4. **模型部署**：训练后量化 / 压缩模型，适配智能戒指的低算力场景。

### 2. 分场景模型选择

#### 2.1 睡眠分期识别

睡眠分期通常分为 5 类（清醒 W、浅睡 N1、浅睡 N2、深睡 N3、快速眼动 REM），核心是**时序分类问题**，推荐以下模型：

方案 1：**LightGBM/XGBoost（梯度提升树）**

- **适用原因**：
    - 无需复杂的时序建模，通过提取时间窗口内的统计特征（如 5 分钟内心率均值、HRV 的 SDNN、呼吸频率方差）即可建模；
    - 对缺失值、非线性特征鲁棒，训练速度快，可解释性强（能输出特征重要性，比如 “HRV 低频功率” 对深睡识别最关键）；
    - 适合中小规模数据集（智能戒指采集的个人数据量通常不大）。
- **特征设计**：
    - 基础统计特征：每个指标的均值、中位数、标准差、最大值 / 最小值、斜率（趋势）；
    - HRV 专项特征：SDNN（正常 RR 间期标准差）、RMSSD（相邻 RR 间期差值均方根）、LF/HF（低频 / 高频功率比）；
    - 时间特征：夜间时段（如 23:00-02:00、02:00-05:00）。

方案 2：**TCN（时间卷积网络）/LSTM（长短期记忆网络）**

- **适用原因**：
    - TCN 是专门处理「时间序列」的卷积神经网络，擅长从连续信号里抓长期规律、短期变化。
    - 直接处理原始时序数据，无需手动提取过多特征，能捕捉心率、呼吸等指标的动态变化（如 REM 期心率波动大）；
    - TCN 相比 LSTM，训练速度更快、对长序列依赖捕捉更好，更适合智能设备端部署。
- **适用场景**：数据集规模较大（如超过 1000 个睡眠夜数据），或需要更高识别精度。

> 3. XGBoost 不推荐（比 LightGBM 慢，且对小样本适配性差）。

> [!warning] TCN 为什么特别适合「智能戒指睡眠分期」？
> TCN 是专门处理「时间序列」的卷积神经网络，擅长从连续信号里抓长期规律、短期变化。
> 你的输入是：**心率、HRV、呼吸频率、血氧、体动（ACT）**
> 全部都是：**连续、时序、低频、慢变化**信号。
> 
> 睡眠分期的特点：
> 
> 1. 每个阶段持续**几分钟～几十分钟**
> 2. 阶段之间**渐变**，不是突变
> 3. 依赖**长期趋势**，不只是某一秒的值
    > - 深睡：心率低、HRV 低、体动极少
    > - REM：心率波动大、体动少
    > - 清醒：体动多、心率高
> 
> TCN 完美匹配：
> 
> - 能抓**长期趋势**（深睡、浅睡、REM 的长规律）
> - 能抓**瞬间变化**（醒了、翻身、做梦）
> - 计算量比 LSTM 小很多
> - **非常适合端侧部署（智能戒指 / 手表）**

> [!info]- TCN 在智能戒指上识别睡眠分期的完整工作流程
> #### 1. 输入（智能戒指采集）
> 每 5s / 10s / 30s 一条数据：
> 
> - 心率 HR
> - 心率变异性 HRV（RMSSD/SDNN）
> - 呼吸频率 RR
> - 血氧 SpO2
> - 体动 ACT（加速度计）
> 
> 构成一个**时序矩阵**：
> 
> [时间步 × 5 个特征]
> 
> 比如 120 个时间步 = 1 小时信号
> 
> #### 2. TCN 做的事情
> 
> TCN 一层一层提取**睡眠相关特征**：
> 
> ##### 浅层（短时序）
> 
> - 心率突然上升 → 可能要醒
> - 体动突然变大 → 清醒
> - 呼吸突然变快 → 做梦（REM）
> 
> ##### 中层（中时序）
> 
> - 心率持续走低 → 进入浅睡 / 深睡
> - HRV 持续下降 → 深睡概率升高
> - 血氧平稳 → 睡眠稳定
> 
> ##### 深层（长时序）
> 
> - 过去 20 分钟心率一直很低 + 几乎不动 → **深睡**
> - 过去 15 分钟心率波动大 + 几乎不动 → **REM**
> - 过去 10 分钟心率高 + 频繁体动 → **清醒**
> 
> #### 3. 输出
> 
> TCN 最后接一个全连接层 + Softmax
> 
> 输出 5 个概率：
> 
> - W 清醒
> - N1 浅睡
> - N2 浅睡
> - N3 深睡
> - REM 快速眼动
> 
> 取最大概率 → 判定当前睡眠阶段。

 #### 2.2 女性生理周期识别

生理周期分为卵泡期、排卵期、黄体期、月经期，核心是**周期型时序分类问题**（周期约 28 天），推荐模型：

方案 1：**LSTM + CatBoost 混合模型**

- **技术背景**：训练阶段可积累足量数据，但推理阶段只能用单日数据快速识别，且用户对耗时敏感。
- **核心思路**：训练阶段用多日时序数据学习周期规律，推理阶段将单日数据 “锚定” 到训练好的周期特征空间中，实现快速分类。
- **适用原因**：
    - 适配 “单日快速识别” 核心需求：推理阶段仅需单日数据，满足用户低等待阈值的体验要求；
    - 训练阶段复用长时序价值：LSTM 可从多周期历史数据中自动捕捉 HRV、皮温的跨天趋势规律，弥补纯 CatBoost 依赖人工特征的短板，提升模型精度。

> [!info]- LSTM + CatBoost 混合模型在女性生理周期识别任务上的工作流程
> #### 1. 训练阶段：提前学习 “周期特征模式”（核心改造）
> 
> 训练阶段依然用用户积累的多周期数据（如 3-6 个周期的每日数据），但重点不是学习 “时序序列”，而是学习 “单日特征组合” 与 “周期阶段” 的映射关系，同时预计算每个用户的**个性化周期基线**：
> 
> - **步骤 1：预计算用户个性化基线**
> 
    > 对每个用户的历史数据，统计各周期阶段的生理指标基线（如：黄体期皮温均值、排卵期 HRV 均值、月经期活动度均值），存储为 “用户基线表”：
> 
    > 表格
> 
    > |周期阶段|皮温基线 (℃)|HRV 基线 (ms)|呼吸频率基线 (次 / 分)|活动度基线|
    > |---|---|---|---|---|
    > |月经期|36.2|45|18|85|
    > |卵泡期|36.4|55|17|90|
    > |排卵期|36.5|70|16|95|
    > |黄体期|36.7|60|17|88|
> 
> - **步骤 2：训练阶段构建 “单日特征 + 周期锚定特征”**
> 
    > 基于历史每日数据，为每一条单日数据构建 “绝对特征 + 相对基线特征”（无需时序序列），作为模型输入：
> 
    > 表格
> 
    > |特征类型|具体特征（仅需单日数据计算）|作用说明|
    > |---|---|---|
    > |单日绝对特征|当日皮温 / HRV / 呼吸频率 / 血氧 / 活动度均值|基础生理状态|
    > |相对基线特征|当日皮温 - 对应阶段基线、当日 HRV - 对应阶段基线|量化单日指标与各周期基线的差异|
    > |周期锚定特征|距上次经期天数（用户手动 / 历史记录）|锚定周期位置（核心！替代时序序列的关键）|
    > |衍生特征|当日皮温昼夜差、HRV 变异率、呼吸频率标准差|放大周期阶段的生理差异|
> 
> - **步骤 3：用 CatBoost 训练 “单日特征→周期阶段” 模型**
> 
    > 放弃 LSTM 的时序输入（推理阶段用不上），直接用上述 “单日特征集” 训练 CatBoost，训练时保留特征重要性输出（保证可解释性）。
> 
    > ✅ 关键：训练阶段已通过历史数据学习到 “距上次经期第 10 天 + 皮温 36.5℃+HRV70ms→排卵期” 这类规律，推理阶段只需输入单日特征即可匹配。
> 
> #### 2. 推理阶段：仅用单日数据快速识别（核心：实时、短输入）
> 
> 推理阶段用户仅需提供**当日生理数据 + 距上次经期天数**（后者可通过用户手动记录 / 历史周期自动推算）
> 
> #### 3. 关键工程细节
> 
> 1. **数据预处理**：
    > - 缺失值：采用线性插值（时序数据）或同窗口均值填充；
    > - 异常值：用 3σ 法则剔除（如皮温突然超过 40℃）；
    > - 时间窗口：睡眠分期用 5 分钟窗口，生理周期用日级窗口。
> 2. **模型轻量化**：
    > - 若部署到智能戒指端，将训练好的 LightGBM/TCN 模型量化为 ONNX 格式，减少模型体积和计算量；
    > - 优先选择树模型（LightGBM/CatBoost），其推理速度远快于深度学习模型。
> 3. **标注数据**：
    > - 睡眠分期：需结合心率、HRV、皮肤温度、呼吸频率、血氧饱和度等标注的标准数据；
    > - 生理周期：需用户手动标注经期起始时间，结合模型自学习优化。

#### 2.3 压力指数计算

**参考玖治科技发明专利**
这份发明专利提出的心理压力测量技术方案，核心是**基于生理数据分布类型适配标准化方式**，解决了传统统一标准化导致的数据偏差问题，大幅提升心理压力预测的准确性，可部署于可穿戴设备实现实时测量，整体涵盖**数据采集、分布判断、差异化标准化、压力预测**四大核心环节，同时配套了装置、计算机可读介质及电子设备实现方案落地，以下是详细归纳：

##### 一、核心设计思路

摒弃传统对生理数据统一采用正态分布标准化的方式，先判断用户心率、心率变异性（HRV）等核心生理数据的分布类型（正态 / 非正态），再匹配对应的标准化方式，确保标准化后生理数据特征分布统一，减少数据偏差对压力预测结果的干扰，同时引入时间衰减、偏差修正等机制优化数据处理，最终通过轻量化模型实现心理压力的精准预测。

##### 二、关键生理数据采集与预处理

1. **采集数据类型**：主要采集光电容积图（PPG）信号、加速度计（ACC）信号，基于 PPG 计算心率（HR）、心跳间期（NN/RR 间期）及 HRV 时域特征（如 rmssd），ACC 信号用于判断用户状态。
2. **静息态数据提取**：将 PPG/ACC 数据分段（如 6s / 段，重叠 5s），计算每段 ACC 数据标准差，若小于阈值（如 0.1g）则判定为静息状态，仅提取静息态下的生理数据用于后续分析（心理压力检测基于静息态数据）。
3. **异常数据处理**：对超出静息态合理范围的 HR（[31,130]）、rmssd（[1,200]）采用**截断方式**修正（超出范围则取边界值），也可选择丢弃；同时对统计分布数据做**防溢出处理**（数据量超阈值则整体缩小 10%），避免存储与计算异常。
4. **采集优化**：检测环境光线强度，强光下添加滤光片优化 PPG 采集；根据用户肤色、运动状态切换绿光 / 红光传感器，适配不同场景提升数据准确性。

##### 三、基于数据分布类型的差异化标准化方式

标准化方式分为**第一标准化方式（非正态分布）**和**第二标准化方式（正态分布）**，核心处理对象为静息态提取的 HR、rmssd 等当前静息特征。

 **（一）第一标准化方式：非正态分布数据转换为标准正态分布**

为核心创新点，通过多步骤将非正态生理数据映射为标准正态分布（均值 0、标准差 1），消除个体生理差异导致的分布不同问题，步骤如下：

1. **更新统计分布**：将当前静息特征匹配到历史静息特征的统计分布特征区间，引入**时间衰减因子**融合历史与当前数据量，更新特征区间数据量（降低久远历史数据的权重，贴合用户当前生理状态）。
2. **计算累积分布特征**：基于更新后的统计分布，为各特征区间分配**时间衰减权重**（采集时间越久权重越低），通过加权求和计算当前静息特征的累积分布占比（第一加权数据 / 第二加权数据），该累积分布符合标准正态分布累积分布特征。
3. **查询正态分布表 + 偏差修正**：根据累积分布特征查询标准正态分布表得到候选标准化数据；将候选数据与设定数量的历史标准化数据组成数据集，若其统计特征（均值、标准差）与标准正态分布误差超预设范围，生成**偏差修正因子**修正候选数据，得到最终标准化生理数据。

**（二）第二标准化方式：正态分布数据常规标准化**

直接采用传统标准化方法，根据正态分布统计参数（均值 μ、标准差 σ）或用户生理数据统计参数（最大值、最小值）处理，如公式\(x'=(x-\mu)/\sigma\)、\(x'=(x-min)/(max-min)\)。

##### 四、心理压力预测实现

1. **特征优化**：可选结合生理数据采集时间、用户所处环境、生理数据频域特征生成**特征标签**，与标准化生理数据共同作为预测输入，提升准确性。
2. **轻量化模型训练与预测**：
    - 训练阶段：将标准化后的 HR、rmssd 作为二维特征，结合人工标注的标准心理压力值，训练小型回归模型（如二元二次多项式回归、线性回归、SVM、轻量树模型等），模型占用内存少、计算量小，适配可穿戴设备。
    - 预测阶段：对用户新的静息态生理数据，重复 “特征提取 - 分布更新 - 标准化” 流程，将得到的标准化数据输入训练好的模型，输出当前心理压力值。
3. **动态优化**：随着用户静息态生理数据的持续累积，统计分布会更贴合用户实际生理特征，标准化结果与压力预测准确性会逐步提升，支持**全天实时动态预测**。

## 4.5 数据安全与隐私需求

### 4.4.1 用户授权机制

- **明确同意：** 每项数据收集和使用都需用户明确授权
- **granular控制：** 用户可单独授权心率、体温、睡眠等各数据项
- **数据导出：** 用户可随时导出自己的所有数据
- **数据删除：** 提供账户注销和数据彻底删除功能
- **分享控制：** 用户可自主决定是否与医生/伴侣分享数据

### 4.4.2 数据加密技术

1. **传输层加密**
   - 使用TLS 1.3协议确保数据在传输过程中的安全性
   - 证书链验证，防止中间人攻击
   - 完美前向保密（PFS），即使密钥泄露也不影响历史数据安全

2. **存储层加密**
   - 设备本地数据采用AES-256-GCM加密
   - 密钥派生使用PBKDF2或Argon2算法，增加暴力破解难度
   - 云端存储使用服务器端加密（SSE）和客户端加密（CSE）双重保护

3. **密钥管理**
   - 使用硬件安全模块（HSM）或安全元件（SE）保护主密钥
   - 密钥分层管理，不同数据类型使用不同加密密钥
   - 定期密钥轮换，降低密钥泄露风险

### 4.4.3 GDPR合规要点

1. **数据最小化原则**
   - 仅收集实现产品功能所必需的数据
   - 明确数据保留期限，到期自动删除
   - 提供数据清理工具，用户可随时删除历史数据

2. **透明性原则**
   - 清晰的隐私政策，用通俗易懂的语言说明数据使用方式
   - 数据处理活动记录，可审计可追溯
   - 定期向用户发送隐私更新通知

3. **用户权利保障**
   - **访问权：** 用户可查看自己的所有数据
   - **更正权：** 用户可更正不准确的数据
   - **删除权（被遗忘权）：** 用户可要求删除个人数据
   - **限制处理权：** 用户可限制数据处理范围
   - **数据可携带权：** 用户可获取结构化、机器可读的数据副本
   - **反对权：** 用户可反对特定的数据处理活动

4. **数据保护影响评估（DPIA）**
   - 对高风险数据处理活动进行DPIA
   - 识别和评估隐私风险
   - 制定风险缓解措施

5. **数据泄露应急预案**
   - 建立数据泄露监测和响应机制
   - 72小时内通知监管机构和受影响用户（如适用）
   - 定期进行安全审计和渗透测试
